# CUMCM Paper Standard

本页是 MathModel Skill 的正式论文范式说明。它不替代建模和代码运行，只约束证据门禁通过后的论文结构、章节展开、图表解释和 Word 排版目标。

## 总体结构

正式论文统一采用 `1 / 1.1 / 1.1.1` 编号：

```text
题目
摘要
关键词

1 问题重述
1.1 背景说明
1.2 题目任务
1.3 本文解决思路概述

2 问题分析
2.1 问题一分析
2.2 问题二分析
...

3 模型假设
4 符号说明

5 模型的建立与求解
5.1 问题一模型的建立与求解
5.1.1 建模思路
5.1.2 变量定义与公式推导
5.1.3 求解算法
5.1.4 结果分析
5.1.5 模型检验或灵敏度分析

5.2 问题二模型的建立与求解
...

6 模型检验与灵敏度分析
7 模型评价与推广
8 参考文献
附录
```

## 字数与密度

- 设结构化子问题数量为 `Q`，正式 outline 的有效字符目标为：最低 `6500 + 1200Q`、理想 `7500 + 1500Q`、建议上限 `10000 + 2200Q`。
- 动态目标是 MathModel Skill 的完整度启发式，不是竞赛官方固定字数规则；赛题或主办方另有要求时以官方规则为准。
- 低于动态最低值的正式格式检查失败；超过建议上限只给 warning，但必须检查重复、空泛扩写或图表解释堆叠。
- 禁止复制段落后只修改数字、序号、连接词或少量同义词来满足篇幅目标。

## 摘要

摘要目标 `800-1200` 字，必须按子问题展开：

- 第一段概括背景、研究对象和总体目标。
- 每个子问题至少说明模型、算法和关键结果。
- 最后一段说明模型检验、灵敏度、评价和推广。
- 关键词 `4-6` 个，覆盖模型方法、算法和问题领域。

## 正文章节

问题重述不能复制题面，应压缩背景、提炼规则、按编号说明任务。问题分析必须逐问写清任务本质、变量和约束、模型选择理由、求解思路、预计输出和验证方式。

第 5 章是正式论文的核心。每个 `5.x` 小节必须包含：

- 建模思路。
- 变量定义与公式推导。
- 目标函数、决策函数或约束条件。
- 求解算法，使用 `Step 1`、`Step 2` 等形式。
- 结果表、结果图、图表解释和结论回扣。
- 模型检验、灵敏度分析或合理性分析。

## 图表、公式与算法

- 每张图表必须先在正文中引用，再插入图表，图表后必须有解释段。
- 图题格式建议为 `图5.1.1 xxx`，表题格式建议为 `表5.1.1 xxx`。
- 公式前必须定义变量，公式后必须解释每一项含义，并说明公式如何进入求解。
- 正式 DOCX 中的公式必须是可编辑的 Word OMML，不能用截图或纯文本公式冒充。
- 算法不能只写“使用 Python 求解”，必须写清输入、状态变量、遍历或优化步骤、停止条件和输出。

## 评价、参考文献与附录

模型评价必须包含优点、不足、改进方向和推广应用。参考文献建议 `5-8` 条，至少覆盖数学建模、概率统计、运筹优化、算法或题目相关领域；文末每条文献必须在正文引用，正文引用编号必须能在文末找到。附录应说明代码位置、结果契约、图表表格索引，必要时给出关键代码片段。

## 最终门禁

- evidence gate 必须重新计算建模脚本、输入和输出哈希，拒绝过期运行记录、空或非有限指标、缺失/空/失败/占位图表表格。
- format gate 必须检查动态篇幅、结构、图表引用、原生公式、正文引文、重复内容和内部工程话术。
- 最终交付必须运行 `check_paper_format.py --render required`，确认 DOCX 可转换为非空 PDF，页数有效且正文文本可提取。
