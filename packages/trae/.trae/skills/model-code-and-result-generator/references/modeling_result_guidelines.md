# Modeling Result Guidelines

本参考文件用于指导 Agent 将数学建模结果沉淀为结构化证据，而不是直接把临时计算过程写进正文。

## 必须沉淀的证据

每个 `question_id` 至少应尽量提供以下信息：

- 模型输出：预测值、最优方案、评价得分、分类结果、聚类标签、仿真曲线摘要等。
- 指标：误差、得分、目标函数值、约束满足率、稳定性指标、敏感性指标等。
- 表格：参数表、结果表、误差表、模型对比表、敏感性分析表或排序表。
- 结论：一句能直接回答原问题的结构化结论。

## 使用边界

`build_result_contracts.py` 第一版只生成结果证据骨架和基础数据画像。真实赛题仍应由 Agent 根据当前数据字段和模型路线生成专用建模代码，并把真实结果写回这些契约。

不得把 `status=template`、`status=draft` 或 `evidence_status=needs_real_modeling` 的内容伪装成最终比赛结论。
