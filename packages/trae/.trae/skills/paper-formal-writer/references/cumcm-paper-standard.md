# CUMCM 正式论文范式

本规范用于 MathModel Skill 的正式成稿阶段。它约束 Agent 写作结构，不替代模型计算和真实结果生成。

## 总体结构

正式论文采用 `1 / 1.1 / 1.1.1` 编号：

```text
题目
摘要
关键词

1 问题重述
1.1 背景说明
1.2 题目任务
1.3 本文解决思路概述

2 问题分析
2.1 问题一分析
2.2 问题二分析
...

3 模型假设
4 符号说明

5 模型的建立与求解
5.1 问题一模型的建立与求解
5.1.1 建模思路
5.1.2 变量定义与公式推导
5.1.3 求解算法
5.1.4 结果分析
5.1.5 模型检验或灵敏度分析

5.2 问题二模型的建立与求解
...

6 模型检验与灵敏度分析
7 模型评价与推广
8 参考文献
附录
```

## 篇幅与内容密度

- 设结构化子问题数量为 `Q`，outline 的有效字符目标为：最低 `6500 + 1200Q`、理想 `7500 + 1500Q`、建议上限 `10000 + 2200Q`。
- 动态目标是本项目的完整度启发式，不是竞赛官方固定字数规则；实际写作优先服从赛题要求、证据密度和推导完整性。
- 低于动态最低值时正式格式检查失败；超过建议上限时给出 warning 并检查冗余。
- 禁止复制段落后只改数字、序号、少量同义词或连接词来满足篇幅目标。

## 摘要

摘要目标 `800-1200` 字，必须按子问题展开：

- 第一段：背景、研究对象、总体目标。
- 后续段落：每个子问题分别说明模型、算法、关键结果。
- 最后一段：模型检验、灵敏度、推广或应用价值。
- 关键词 `4-6` 个。

## 问题重述

问题重述不能复制题面。必须：

- 压缩背景。
- 提炼规则或约束。
- 按子问题编号说明本文要解决的任务。

## 问题分析

每个子问题单独分析，必须说明：

- 任务本质。
- 主要变量与约束。
- 模型选择理由。
- 求解思路。
- 预计输出和验证方式。

## 模型建立与求解

每问至少包含：

- 建模思路。
- 变量定义与公式推导。
- 目标函数或决策函数。
- 约束条件。
- 求解算法和 Step 步骤。
- 结果表与结果图。
- 图表解释。
- 结论回扣。
- 检验、灵敏度或合理性分析。

## 图表与公式

- 每张图表必须在正文中被引用。
- 图表前必须有引导句，图表后必须有解释段。
- 图题、表题必须清楚说明图表用途。
- 公式前定义变量，公式后解释每一项含义。
- 正式 DOCX 中的公式必须转换为可编辑 Word OMML，不能使用截图或纯文本公式冒充。
- 算法步骤使用 `Step 1`、`Step 2` 等格式。

## 模型评价与参考文献

模型评价应包含：

- 模型优点。
- 模型不足。
- 改进方向。
- 推广应用。

参考文献建议 `5-8` 条，包含数学建模、概率统计、运筹优化、算法或相关领域资料。每条文献必须在正文实际引用，正文引用编号也必须在文末存在。

## 正式交付门禁

- evidence gate 必须为 PASS，且报告记录的输入哈希与当前结果、指标、图表、表格和运行记录一致。
- format gate 必须拒绝控制字符、重复或数字归一化后近重复段落、附录前的内部工程话术和缺失的原生公式。
- 最终交付运行 `check_paper_format.py --render required`，要求 DOCX 能通过 LibreOffice 转为非空 PDF，并获得有效页数和可提取文本。
